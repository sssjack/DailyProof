# DailyProof 设计文档

## 产品定位

DailyProof 对标 ToDo，但核心不是简单待办，而是把“月度目标 -> 每日计划 -> 倒计时执行 -> 分类记录 -> 可视化复盘”串成闭环。新版没有独立题库入口，而是把“刷题”作为每日计划中的一种任务类型来记录。

## 目标用户

- 普通用户：创建月计划，自动拆分每日任务；事项可直接勾选，刷题需填写题量、正确率、用时和复盘备注。
- 管理者：查看多用户活跃、任务完成率、刷题量、平均正确率、用时和刷题结果明细。

## 核心页面

- 首页：哲学感、科技感第一屏，表达“把每天变成证据”，并提供系统使用说明。
- 登录注册：支持普通注册、默认管理员和体验账号登录。
- 工作台：展示今日任务、事项/刷题类型、倒计时、完成打勾、刷题结果录入、月计划创建和 AI 拆分。
- 数据中心：按日、周、月、总维度汇总刷题量、正确率、用时趋势和标签分布。
- 管理后台：管理员查看用户进度和刷题结果记录。

## 视觉设计

- 气质：科技感、呼吸感、时尚但克制。
- 字体：标题使用衬线艺术字体栈，界面正文使用现代无衬线字体栈。
- 主题：黑夜、明亮、脉冲三套主题可切换。
- 布局：工作台信息密度高，任务优先，不做营销式空泛堆叠。

## 使用流程

1. 用户登录系统。
2. 创建月计划，填写目标、目标用时、正确率和每日时间安排。
3. 系统调用 DeepSeek 配置进行计划拆分；失败时使用内置规则兜底。
4. 用户每日按任务打勾或启动倒计时。
5. 倒计时到达目标时，事项任务自动完成；刷题任务提示用户填写刷题结果后再完成。
6. 刷题任务必须补充题量、正确率、用时分钟数和可选复盘备注。
7. 数据中心按标签统计当天平均正确率、刷题量和用时，并展示趋势、周统计、月统计和总统计。
8. 管理后台读取刷题结果汇总。

## 默认计划示例

- 07:00-08:00 数量关系 10 道 + 复盘
- 18:00-20:30 健身 / 吃饭 / 回家
- 20:00-21:00 资料分析套题 1 + 复盘
- 21:00-22:00 资料分析套题 2 + 复盘
- 22:00-23:00 夜间学习 + 睡前整理

目标：资料分析每套 27 分钟内，正确率达到 90%。

## 刷题标签

- 资料分析：`data_analysis`
- 数量关系：`quantitative`
- 言语理解：`verbal`
- 图推：`graphic_reasoning`
- 判断推理：`judgement_reasoning`
- 政治理论：`political_theory`
- 常识：`common_sense`
